Learning Points

  1. Bosh is very powerful and does amazing things
  2. Your team needs to learn all the things

Stage 1: Denial

“the survivor imagines a false, preferable reality”
  1. Why are we using a thing that no-one else does?
  2. Is writing manifests and learning so much really the best solution?
  1. Can't be any good, website doesn't look like Terraform's...

Learning Points

Remember that Bosh gives you:

  1. Declaration of intent
  2. Effectively immutable
  3. IaaS agnostic
  4. Highly automated
  5. Self-healing
  6. Open source and extensible(ish)
  1. There isn't an easy answer - just remember what it does for you

Stage 2: Anger

“the survivor becomes frustrated”

Learning Points

  1. Be prepared to read up
  2. You will need to debug into many components

Stage 3: Bargaining

“the survivor seeks compromise”
  1. Subverting Bosh mitigates its benefits - eg using Chef, replacing Monit

Learning Points

  1. Do not argue with Bosh

Stage 4: Guilt

“the survivor believes to have committed sin”
  1. Boshed infrastructure is supposed to last
  2. Unlike other tools - everyone else must scale the cliff

Learning Points

  1. Same as any niche technology choice
  2. Radiate reasons for using it
  3. Pair on release development

Stage 5: Depression

“the survivor has feelings of severe despondency and hopelessness”

Learning Points

  1. Tooling is not great, do not expect it
  2. Use bosh-lite
  3. Consider a pool of Boshes for CI

Stage 6: Acceptance

“a calm, retrospective view for the survivor, and a stable set of emotions”
  1. Stockholm syndrome - everyone over the cliff loves it
  2. Fundamental philosophical truths (declarative, immutable)
Must be good if it has a parody

Learning Points

You now have:

  1. IaaC
  2. A team capable of end-to-end continuous delivery

Stage 7: Hope

“the survivor develops and a positive attitude about the future”
  1. Joke: off-by-one errors
  2. bosh.io
  3. CCP allows zero-cost trial of releases, and downloading of them

Bosh Self-Help Guide

  • You can't do Bosh half-arsedly
  • It won't be easy, but it will be worth it
  • Be willing to become poly-skilled
  • Exploit bosh-lite for testing
  • Acceptance-test releases
